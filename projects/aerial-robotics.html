AUTONOMOUS SYSTEMS

Aerial Robotics Competition System

Autonomous quadrotor navigation and robust computer vision system for a final aerial robotics tournament.

Overview

This project was the final integration effort for a semester-long aerial robotics course focused on autonomous quadrotor systems. The final tournament was split into two major challenges: an autonomous navigation task and a separate computer vision task. The navigation side required the simulated quadrotor to plan through an obstacle course, interact with target balloons, and land. The vision side required detecting real balloon targets in images that also contained false balloon-like objects.

Our team placed 4th overall in the class. My main contribution was on the vision side, where I adapted the earlier balloon localization pipeline to be more robust to false positives by implementing and tuning RANSAC-based outlier rejection before the final 3D localization step.

Project Summary

Final tournament system combining autonomous navigation, A* path planning, trajectory pruning, simulated quadrotor execution, computer vision, RANSAC outlier rejection, and 3D balloon localization.

Autonomous Navigation System

The navigation portion of the tournament used a layered autonomy structure. At a high level, the system planned the mission objective, generated a global path through the environment, selected useful local path segments, converted those segments into feasible trajectories, and passed those commands to the simulated quadrotor controller.

A major challenge was balancing path quality, computational efficiency, and trajectory feasibility. The system could not simply follow every grid node from the planner because dense waypoint sequences made the polynomial trajectory solver less effective. Path pruning helped reduce unnecessary waypoints, but overly aggressive pruning could make trajectories less safe near obstacles. This created a real engineering tradeoff between speed, smoothness, and obstacle clearance.

Navigation Pipeline

Mission objective → A* global path → path pruning → local trajectory generation → trajectory validation → quadrotor execution → target interaction → landing

Vision Test & RANSAC Detection System

The vision test was a separate portion of the tournament. It did not rely on the path planning system. Instead, the goal was to detect balloon targets from image data and estimate their 3D locations using camera measurements and known camera poses.

The baseline localization method worked well when the image detections were clean, but the tournament images included false balloon-like objects that could pass the initial color filtering step. These false positives created outlier measurements, which could skew the final 3D estimate if all detections were treated as valid.

To address this, I implemented RANSAC-based robust estimation on top of the earlier weighted least-squares structure computation. The RANSAC logic selected candidate measurement subsets, checked which detections were geometrically consistent with the candidate solution, rejected outliers, and recomputed the final 3D balloon estimate using the best support set.

Vision Result

RANSAC significantly improved localization on the tournament-train image set. For the blue balloon case, the reported localization error dropped from roughly meter-scale error without RANSAC to centimeter-scale error with RANSAC. The red balloon case also improved after outlier rejection.

Technical Foundation

The final tournament system was built on several technical pieces developed earlier in the course. The project began with a MATLAB quadrotor simulator using Newton-Euler rigid-body dynamics and rotor thrust/torque models. The simulator was later refined with aerodynamic drag, first-order motor dynamics, and a two-layer controller for trajectory and attitude tracking.

The autonomy stack was then expanded with simulated GNSS, camera, and IMU measurements, along with UKF-based state estimation. Planning and trajectory generation were developed using DFS, Dijkstra, A*, and polynomial trajectory generation. The perception pipeline was built from OpenCV balloon detection and weighted least-squares 3D feature localization before being upgraded with RANSAC for the final tournament vision test.

Engineering Takeaways

This project showed me that autonomous systems are not just a collection of individual algorithms. The hard part is making planning, control, estimation, and perception work together under realistic constraints. The final tournament made this especially clear because the navigation system had to be fast and feasible, while the vision system had to be robust to bad measurements.

The most valuable lesson was learning how to think in terms of system integration. A good path is not useful if it cannot be converted into a trackable trajectory, and a detected object is not useful if false positives corrupt the final 3D estimate. This project helped me understand autonomy as a full engineering pipeline rather than a set of isolated class topics.
