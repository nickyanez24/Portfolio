AUTONOMOUS SYSTEMS

Aerial Robotics Autonomy Stack

Simulated quadrotor autonomy system combining dynamics, controls, estimation, path planning, trajectory generation, computer vision, and robust 3D localization.

Overview

This project developed a simulated aerial robotics autonomy stack for autonomous quadrotor flight. Over the semester, the system grew from a baseline MATLAB dynamics simulator into a larger autonomy workflow involving feedback control, high-fidelity sensor simulation, state estimation, graph-based planning, smooth trajectory generation, computer vision, and final competition integration.

The final tournament combined autonomous navigation with a computer vision detection challenge. Our team developed a planning architecture for obstacle-course traversal and implemented a robust vision pipeline that used RANSAC to reduce the effect of false balloon detections in an adversarial image environment.

Final Competition Challenge

The final competition required an autonomous quadrotor system to complete two major tasks: navigate through an obstacle course and perform a computer vision detection test in a challenging environment. The autonomy side required planning feasible paths through obstacles, generating trackable trajectories, and coordinating the vehicle through multiple mission objectives. The vision side required detecting balloon targets and estimating their 3D locations while rejecting false measurements.

Rather than treating these as disconnected pieces, our team approached the tournament as a full autonomy architecture. The system was organized around mission planning, global planning, local planning, and a monitoring/orchestration concept that could support replanning if the environment changed.

Our team placed 4th overall in the class, reflecting the combined performance of our autonomous navigation strategy and robust computer vision pipeline.

Project Framing

The most important outcome of this project was not one individual algorithm. It was the integration of simulation, controls, estimation, planning, and perception into a single autonomous quadrotor workflow.

System Architecture

The autonomy stack can be viewed as a pipeline from mission-level objectives to vehicle-level control. The mission planner selected the order of objectives, the global planner generated a path through the map, the local planner converted path segments into smooth polynomial trajectories, and the controller tracked the resulting reference trajectory using the simulated quadrotor model.

Autonomy Pipeline

Mission objectives → global path planning → local trajectory generation → state estimation → feedback control → perception updates → final competition execution

Simulation & Control

The project began with a MATLAB quadrotor simulator based on Newton-Euler rigid body dynamics. The simulator propagated position, velocity, attitude, and body rates while modeling rotor thrust and torque inputs. After validating the baseline dynamics implementation, the simulator was refined to include aerodynamic drag and first-order motor dynamics, making the vehicle response more realistic.

I then implemented closed-loop control using a two-layer structure: a trajectory controller that commanded total thrust and desired body-axis orientation, and an attitude controller that commanded body torques. This allowed the quadrotor to track reference trajectories such as circular flight paths while maintaining the intended yaw behavior.

State Estimation

After the control system was working with true-state feedback, the project was extended to simulate realistic onboard sensing. The sensor suite included GNSS measurements, camera measurements, and IMU accelerometer and gyro measurements. A UKF was then used to estimate the vehicle state, including position, velocity, attitude error, and IMU biases.

This step made the simulation more representative of an actual autonomous vehicle, where the controller cannot directly use the true simulated state. Instead, the controller had to rely on the estimated state produced by the sensor fusion pipeline.

Planning & Trajectory Generation

The planning portion connected discrete graph search with continuous trajectory generation. I implemented and compared DFS, Dijkstra, and A* in C++, using occupancy grids to study path cost, nodes explored, and runtime. DFS could find valid paths but did not guarantee optimality, while Dijkstra and A* provided lower-cost paths by accounting for cumulative path cost and heuristic guidance.

Since a quadrotor cannot directly track a list of discrete grid nodes, the path planning output was passed into a polynomial trajectory generator. The polynomial solver converted waypoints into smooth position, velocity, and acceleration histories that could be tracked by the MATLAB quadrotor controller. This created a full-stack planning workflow from obstacle map to trackable flight command.

Computer Vision & 3D Localization

The perception portion estimated the 3D location of target objects from multiple 2D camera measurements with known camera poses. The structure estimation method used a weighted least-squares formulation to solve for the unknown feature location and estimate uncertainty.

I also worked with an OpenCV-based balloon detection pipeline. Images were filtered in HSV color space, cleaned using morphological operations, and processed with contour detection to estimate balloon centers in image coordinates. These detected 2D centers were then passed into the 3D structure estimation pipeline.

Robust Vision with RANSAC

For the final competition, our team improved the original weighted least-squares structure computation by adding RANSAC. This made the final 3D balloon estimate less sensitive to false detections by identifying a consistent support set of measurements before computing the final target location.

This was especially important because the vision test included an adversarial environment where some detections could pass through the earlier image-processing filters but still correspond to false balloon measurements. RANSAC helped separate the reliable measurements from the outliers before estimating the final 3D position.

Results & Validation

The full project demonstrated the major components of a simulated quadrotor autonomy system. The simulator and controller were validated through trajectory tracking, the estimator was tested with noisy sensor measurements, the planner generated paths through occupancy grids, the polynomial solver converted those paths into smooth trajectories, and the vision pipeline estimated target locations from camera images.

In the final tournament, these pieces came together in a competition-style workflow. The autonomy system handled mission planning and trajectory generation, while the vision system used robust estimation to improve performance in the presence of false detections.

Key Result

Built a semester-long quadrotor autonomy pipeline that progressed from rigid-body simulation to final competition integration, connecting control, estimation, planning, trajectory generation, computer vision, and robust 3D localization. Our team placed 4th overall in the class in the final aerial robotics tournament.

Engineering Takeaways

This project helped me understand how different autonomy modules connect in a larger robotic system. A controller depends on a reliable simulator, the simulator becomes more realistic when sensing and estimation are included, a planner must produce trajectories that the controller can actually track, and a perception system must be robust enough to handle imperfect measurements.

The final competition made the project feel closer to a real autonomous systems problem because success depended on integration, not just individual algorithms. The most valuable lesson was learning how to think about robotics as a full pipeline, where dynamics, software, planning, estimation, and perception all affect one another.
